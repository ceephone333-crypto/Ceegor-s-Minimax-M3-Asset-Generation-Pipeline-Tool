v1.1.5: fix output folder regression + asar example-write + CSS hover + %APPDATA% default

Three user-reported regressions squashed in one pass:

1. Generate always says "No output directory set" even after picking one.
   Root cause: `ensureSubDir(name)` (called from imageTab/speechTab/
   musicTab/videoTab) was lost during the Phase 3 Block 29 refactor
   that extracted app.js into 24 section files. The ReferenceError
   from the undefined function landed in the catch block, which
   hard-coded the misleading "No output directory set" toast.
   Fix: restored `ensureSubDir` in renderer/app.js (also exported on
   window so the tab scripts loaded earlier in index.html can see it).
   Also improved the four catch blocks to surface the real error.

2. Exporting the example batch file fails with ENOENT inside the asar.
   The `batches:generateExamples` IPC wrote to `deps.appRoot/example_*.md`,
   where `appRoot = path.resolve(APP_ROOT, '..')`. In a packaged build
   APP_ROOT is inside `resources/app.asar/main`, so `appRoot` resolves
   to `resources/app.asar/` itself — a read-only archive. fs.writeFile
   threw ENOENT and the user couldn't get the example docs at all.
   Fix: write the examples to `cfgMod.effectiveOutputDir(cfg)` (the
   user's actual output folder) with a mkdir-p so it exists on first
   run. Updated the toast to reflect the new destination.

3. Folder explorer: every second row has no hover effect.
   CSS specificity bug: the zebra rule
   `.fb-item:nth-child(even):not(.selected):not(.fb-drop-target)`
   has 4-class specificity and beat the hover rule
   `.fb-item:nth-child(even):hover` (3-class), so the zebra background
   won every time. Fix: mirror the same selector structure
   (including the two :not() guards) on the hover rule so its
   specificity matches and cascade order lets it win.

4. User-requested feature: default output_dir to %APPDATA% when unset.
   Previously the fallback was `<exe-dir>/generated` (for packaged
   builds: `<dist-stable>/win-unpacked/generated`) — an unexpected
   location. Now: `<userData>/generated` (Electron's app.getPath
   ('userData') = %APPDATA%/<productName>). Wired through:
   - src/config.js: new `defaultOutputDir()` helper.
   - main/ipc/registerConfigIpc.js: new `config:defaultOutputDir` IPC.
   - preload.js: bridge.
   - renderer/app.js init(): use the IPC for the fallback instead
     of fabricating a path from configPath().

Tests: 123 pass (was 107; +16 new covering ensureSubDir,
defaultOutputDir, batches:generateExamples destination, and
the asar-vs-output-dir regression guard). 0 lint errors.
